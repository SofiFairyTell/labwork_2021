## Основные понятия 

В рамках информационного контура предприятия имеется и передается информация о целях управления, о состоянии управляемого процесса и о факторах воздействия. 
Информационный контур вместе со средствами сбора, передачи и хранения информации, а также персоналом осуществляющим эти действия с информацией, образуют информационную систему организации.

ИС = Совокупность ИТ + Персонал + ИнфРесурсы

Миссия ИС это производство нужной для организации информации для обеспечения эффективного управления

Субъектами в ИС - ИТ-менеджеры, а объектом - ИС. 

Классификация: 
1. По масштабу: государственные, региональные, отраслевые, отдельных предприятий. 
2. По числу пользователей: персональные (однопользовательские), многопользовательские (распределенные)
3. По отраслевой принадлежности: в промышленности, сельского хозяйства, траспорта, банковские, статистические и т.д. 
4. По выполняемым функциям: бухучета, материально-технического снабжения, документооборота
5. По уровню обслуживания управления: стратегические ИС, оперативного уровня
6. В зависимости от степени автоматизации управленческих функций: локальные, интегрированные 

## Стандарты

1. MPS (Master Planning Scheduling ) Объемное каледарное планирование - заключается в планировании ключевых показателей каждого выпускаемого изделия в привязке к временным отрезкам планирования. 
    Этапы: 

    1. Сначала формируется план продаж с разбивкой по периодам. 
    2. По плану продаж с учетом имеющихся запасов готовой продукции на складе формируется план пополнения запасов
    3. Оцениваются результаты выполнения заявок

    Недостатки: не учитывает экономические параметры. 

2. MRP (Material required planning) планирование потребности в материалах - формирование заказов на комплектующие и материалы, опираясь на данные объемно-календарного плана
    Этапы: 
    1. Расчитать полную потребность с учетом запасов на складе 
    2. Сколько надо докупить: Чистая потребность= Полная потребность - на складе + др.потребности + страховой запас
    3. Написание плана заказа 

3. MRP (manufacturing resourses planning) - планирование не только материалов, но и персонала
4. ERP (enterprise resourse planning) -
    1. Репозиторий - содержит всю информацию накопленную в процессе ведения бизнеса. 
5. CSRP - хранение ресурсов синхронизированно с покупателями. 

## Планирование

Планирование позволяет: 
1. создавать планы ИС, поддерживающие развитие фирмы
2. ориентировать разработчиков на результаты 
3. эффективнее использовать ресурсы ИС
4. закладывать большую управляемость и лучшую интеграцию существующих и будущих ИС

Этапы планирования:
1. Определение цели ИС, которая должна соотвествовать обще-фирменым целям
2. Анализ внешней среды фирмы для выявления угроз и возможностей
3. Анализ внутренней среды предприятия, который включает: 
    1. Анализ технических средств
    2. Анализ ПО
    3. Анализ данных и приложений (информационных потоков)
    4. Анализ кадров
    5. Анализ бюджета
    6. Организация и управление в сфере информатизации 

Технические средства: типы, характерстики, мощности, возраст центральных и децентрализованных ЭВМ. То же для принтеров.
